v0.1.12 — Script API fixes and runtime Redis resilience · Исправления API скриптов и устойчивость runtime к Redis

---

## English

### Fixed

#### Script API
- `PUT /scripts/{id}` with `code` now updates the entrypoint file in DB and object storage (parity with create)
- Script deletion removes related notifications before runs — no more FK constraint errors (HTTP 500)

#### Runtime
- Reconnects to Redis after `ConnectionError`
- Log publishing tolerates transient Redis/backend failures

### Changed

- Backend: SQLAlchemy 2.0.51, Pydantic 2.13.4, pydantic-settings 2.14.2, redis 8.0.1, python-multipart 0.0.32
- Frontend: react-router-dom 7.18.1, @tailwindcss/vite 4.3.2
- CI: actions/checkout v7

### Upgrade

```bash
git fetch --tags
git checkout v0.1.12
docker compose -f docker-compose.yml -f docker-compose.prod.yml up -d --build
```
